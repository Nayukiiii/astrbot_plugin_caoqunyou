# astrbot_plugin_cao群友

AstrBot QQ群插件，群友互草版本。不@就随机挑一个群友，@指定人就草他！

> 此仓库由 [Claude](https://claude.ai) 编写

---

## 功能

- **草群友** — 向群友发起草草（不@随机挑人，@了就草指定人），成功/失败基于密码学真随机判定
- **群友反草** — 草群友失败时，被草目标有概率反草回来，并要求对方选择里面/外面
- **群友体内** — 查看被草群友体内累计注入量及排行榜（月度统计，被草维度）
- **群友战绩** — 查看近30天互草战绩排行（草人注入总量榜 + 被草次数榜）
- **杂鱼排行** — 查看近30天选择外面的杂鱼排行（放弃次数榜 + 放弃总量榜）
- **草群友排行** — 查看群内草群友排行（主动草人榜）
- **草群友关系图** — 可视化群内互草关系网络（有向多对多图）
- **重置草群友** — 重置今日草群友次数（普通用户需掷骰，bot管理员直接重置）

---

## 与草nj版本的核心区别

| 特性 | 草nj | 草群友 |
|------|------|--------|
| 草的目标 | 固定（配置里的nj） | 随机/@ 指定群友 |
| 反草方 | 固定nj反草用户 | 被草的群友反草攻击者 |
| 体内统计 | 以nj为中心 | 以被草群友为中心 |
| 战绩统计 | nj反草注入量 | 草人注入总量 + 被草次数 |
| 关系图 | 所有人→nj 的星形图 | 群友互草 多对多有向图 |

## 自定义配置

### 插件配置项

| 字段 | 说明 | 默认值 |
|------|------|--------|
| `cao_probability` | 草群友触发概率（%） | 30 |
| `fancao_probability` | 反草基础强度（%），设0则关闭反草 | 50 |
| `cao_daily_limit` | 每人每天草群友次数上限 | 5 |
| `reset_daily_attempts` | 普通用户每天重置尝试次数上限 | 3 |
| `allow_self_cao` | 是否允许自草 | false |

### 称号与提示词

插件目录下有六个 JSON 文件，可自行编辑：

| 文件 | 说明 |
|------|------|
| `qy_body_titles.json` | 群友体内排行图的称号标签（按累计量/次数） |
| `qy_battle_titles.json` | 群友战绩排行图的称号标签（按累计量/次数） |
| `outside_titles.json` | 杂鱼排行图的称号标签（按放弃量/次数） |
| `qy_body_comments.json` | 草群友选里面后的提示词（按本次注入量） |
| `qy_battle_comments.json` | 反草选里面后的提示词（按本次注入量） |
| `qy_fancao_comments.json` | 反草成功时的台词（按仇恨值分档） |

## 概率机制说明

完整继承草nj的所有概率机制，参见原版说明：

### 草群友触发

使用操作系统熵源（`secrets` 模块）生成真随机数判定，不可预测、不可复现。

### 注入量

采用**对数正态分布**，日常集中在几十到几百 mL，极限上限 100L。射完后附带评级：

| 评级 | 范围 |
|------|------|
| 雕王 | ≥ 4000 mL |
| 半雕王 | 2000 - 3999 mL |
| 半死不活 | 800 - 1999 mL |
| 杂鱼～ | < 800 mL |

### 反草触发

草群友失败后，被草目标的反草概率由以下四个因子动态计算：

- **Q — 逃脱质量（主导）**：用 sigmoid 函数将"逃走百分比"转换，越险逃反草概率越高
- **G — 累积仇恨**：该用户近30天草该目标次数越多越记仇
- **T — 今日群内压力**：当天群内已被草次数越多，反扑意志越强
- **E — 随机熵扰动**：每次判定注入真随机扰动（±6%）

### 反草后的角色关系

```
原攻击者 → 草群友失败 → 目标反草 → 原攻击者选里面/外面
                                   ↑
                           反草方=原被草目标
                           被反草方=原攻击者
```

## 数据说明

- **群友体内数据**：每月1日0点自动重置
- **草群友/反草/杂鱼战绩数据**：滚动统计近30天记录
- 数据存储在 AstrBot `data` 目录下，不会因插件更新丢失
